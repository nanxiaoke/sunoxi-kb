## 🎯 修复目标
解决搜索索引只包含2个文档的问题，实现完整搜索功能。

## 🔍 问题诊断

### 原始问题
- **现象**: 搜索索引只包含2个文档（`technologies/`目录下的2个旧文档）
- **影响**: 搜索功能不完整，16个wiki文档中只有2个可搜索
- **错误信息**: 搜索优化器显示"索引为空"，所有测试查询返回0结果

### 根本原因分析

#### 1. **文档格式不兼容**
- **旧格式**（AI处理）: 使用表情符号和特定格式
  - 分类: `> **分类**: 教程`
  - 摘要: `## 📝 摘要`
  - 关键点: `## 🔑 关键点`
  - 实体: `## 🏷️ 实体与概念`
  - 原始内容: `## 📄 原始内容`

- **新格式**（简单转换）: 使用纯文本格式
  - 分类: `**分类**: articles`
  - 摘要: `## 摘要`
  - 关键点: `## 关键点`
  - 原始内容: `## 原始内容`

#### 2. **正则表达式不匹配**
- `_extract_document_info()`方法中的正则表达式只匹配旧格式
- 新格式文档无法提取分类、摘要等信息，返回`None`，导致不被索引

#### 3. **索引重建逻辑问题**
- `kb_cli.py`中的`--rebuild`参数未正确传递到搜索系统
- `init_search()`方法总是调用`build_index(rebuild=False)`
- 即使指定`--rebuild`，也只会加载现有索引

#### 4. **统计功能格式问题**
- `command_stats()`方法使用旧格式正则表达式统计分类
- 导致新格式文档分类显示为"未分类"

## 🔧 修复实施

### 1. **修复文档信息提取** (`search.py`)
**修改**: `_extract_document_info()`方法
- 添加对新旧两种格式的支持
- 分类提取: 先尝试新格式`**分类**: xxx`，再尝试旧格式`> **分类**: xxx`
- 摘要提取: 支持`## 摘要`和`## 📝 摘要`
- 关键点提取: 支持编号列表和项目符号两种格式
- 原始内容提取: 支持两种格式

### 2. **修复索引重建逻辑** (`kb_cli.py`)
**修改**: 
- `init_search()`方法: 添加`rebuild`参数，支持强制重建
- `command_search()`方法: 传递`args.rebuild`到`init_search()`

### 3. **修复统计功能** (`kb_cli.py`)
**修改**: `command_stats()`方法
- 更新分类统计正则表达式，支持新旧两种格式
- 添加默认分类处理

### 4. **创建缺失目录**
```bash
mkdir -p ~/karpathy-kb/wiki/config
```

### 5. **重新处理失败文档**
- 重新运行`simple_converter.py`处理所有15个原始文档
- 4个JSON文件成功转换到`wiki/config/`目录
